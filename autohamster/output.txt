### Статьи по автоматизации тестирования
* [Executing visual studio 2010 unit tests without installing visual studio](http://mindinthewater.blogspot.com/2011/02/executing-visual-studio-2010-unit-tests.html) (Прислал(-а): @dzhariy) <br>Самый рабочий документ о том, как запустить MS-Test без установленной Visual Studio 2010<br><br>
* [Thoughts on Test Automation in Agile](http://www.infoq.com/articles/thoughts-on-test-automation-in-agile) (Прислал(-а): @dzhariy) <br>Советы и план действий по внедрению автоматизации. Не знаю как вы, а я уже иду похожем путем, но, например, об идеи создать отдельный бэклог автоматизации, как советует автор, я задумался только сейчас. <br><br>
* [Инструменты тестирования приложений для мобильных устройств: обзор вариантов и возможностей](http://www.enterra.ru/blog/tools_for_qa/)  <br>Обзор инструментов (в том числе и для автоматизации), полезных мобильному тестировщику. Позволит выбрать подходящий под конкретные нужды инструмент.<br><br>


### Инструменты
* [Linq to Excel](https://code.google.com/p/linqtoexcel/) (Прислал(-а): @dzhariy) <br>Одно из самых простых и самых удобных средств прочитать Excel файл для .NET/C#<br><br>
* [BrowserSwarm](http://blogs.windows.com/ie/b/ie/archive/2013/09/26/introducing-browserswarm-less-time-testing-more-innovation-on-the-web.aspx)  <br>Это инструмент будет полезен JavaScript разработчикам. Умеет гонять JavaScript-юнит тесты на различных браузерах одновременно. Будет ли эта штука бесплатной? – Вполне возможно, ведь Майкрософту очень важно то, чтобы проекты тестировались и в IE тоже. <br><br>
* [HTML5 Resource Performance](http://dev.hubspot.com/blog/webkit-resource-performance) (Прислал(-а): @dzhariy) <br>Интерфейсы для сбора статистики по клиентской производительности уже появились в главной ветке Google Chrome. <br><br>


### Видео по автоматизации тестирования
* [TestBash Video: Balancing Test Automation Techniques – Matt Archer](http://www.ministryoftesting.com/2013/04/testbash-video-balancing-test-automation-techniques-matt-archer/) (Прислал(-а): @dzhariy) <br>Начиная где-то с 30-й минуты, докладчик рассказывает, как они сделали инструмент для полу-автоматизированного тестирования, отдав всю нудную работу машине, а интеллектуальную – человеку.   <br><br>
